## ロット設計（日本株現物・単元株制度対応）

### 前提条件
```
取引対象: 日本株現物
1単元 = 100株
株価例: 500円
1tick = 1円/株 → 100円/単元
取引単位: 単元（100株）
```

### 1単元あたり必要資金の計算

#### 損失バッファ
```
想定ワースト損失: 10 tick/本
1tick = 100円/単元
→ 1本あたり損失 = 1,000円

最大連敗: 7本
安全係数: ×3（標準）
→ 損失バッファ = 1,000円 × 7 × 3 = 21,000円
```

#### 建玉拘束資金（現物）
```
株価 × 100株 = 500円 × 100 = 50,000円
```

#### 合計
```
1単元あたり必要資金 = 71,000円
  - 損失バッファ: 21,000円
  - 建玉拘束: 50,000円
```

### 資金100万円の場合の最大単元数
```
理論最大 = 1,000,000 ÷ 71,000 ≈ 14単元
```

### 運用段階別設定

#### Phase 0: ペーパー/超小ロット（1-2週間）
- **目的**: 実スリッページ測定、約定確認
- **単元数**: **5-8単元**（理論の36-57%）
- **評価基準**:
  - 最大連敗 ≤ 10本
  - 実スリッページ < 2 tick/本
  - 約定拒否率 < 5%

#### Phase 1: 小ロット実運用（2-4週間）
- **単元数**: **8-10単元**（理論の57-71%）
- **安全係数**: ×3（損失バッファ 21,000円）
- **評価基準**:
  - 日次DD < 平均日次利益×2
  - 週次勝率 ≥ 60%

#### Phase 2: 標準ロット
- **単元数**: **12-14単元**（理論の86-100%）
- **安全係数**: ×2.5（損失バッファ 17,500円）
- **評価基準**:
  - 月次PF ≥ 1.3
  - 月次平均PnL ≥ 2.0 tick

### 重要な注意事項

#### ❌ 避けるべき間違い
- 1株単位で計算（単元を無視）
- 建玉拘束資金を考慮しない
- 理論最大を超えた運用

#### ✅ 必ず守るべきこと
- 単元（100株）単位で計算
- 現物取引の資金拘束を考慮
